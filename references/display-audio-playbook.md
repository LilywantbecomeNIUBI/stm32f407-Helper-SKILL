# 屏幕与音频补充参考

补充当前 skill 在 `TFTLCD（MCU屏）`、`触摸屏`、`外部 SRAM`、`汉字显示`、`图片显示`、`音乐播放器` 方向的教材联动、例程入口和迁移提醒。

## 使用顺序

遇到这几类任务时，优先按下面顺序查：

1. 先看当前项目和 `.ioc`，确认是否已经有屏幕总线、触摸链路、存储介质或音频链路等底层条件。
2. 再用 `assets/example_catalog.json` 找最接近的本地例程。
3. 再用 `assets/book_catalog.json` 和 `assets/book_excerpt_catalog.json` 找教材章节与关键页。
4. 最后只展开少量关键源码，不要一开始就扫完整个例程目录。

## 教材联动

- 第四十章 `触摸屏实验`
  适合做触摸坐标读取、触摸校准、触摸和显示联调。
  推荐先看页码 `621-645`，重点页 `621 / 626 / 627 / 629`。
  如果是 `实验28 触摸屏实验`，优先把这章作为主教材入口。

- 第五十二章 `汉字显示实验`
  适合做字库加载、中文菜单、中文状态页。
  推荐先看页码 `821-838`，重点页 `821 / 828 / 829 / 830`。
  如果外部 SRAM 准备用来放字库缓存，可以把这章和 `实验36 外部SRAM实验` 一起看。

- 第五十三章 `图片显示实验`
  适合做 BMP/JPEG/GIF 显示、相框、界面素材显示。
  推荐先看页码 `839-850`，重点页 `839 / 840 / 841`。
  如果外部 SRAM 准备用作图片缓存或显示缓冲区，也建议和 `实验36 外部SRAM实验` 联动阅读。

- 第五十五章 `音乐播放器实验`
  适合做 WAV 播放、ES8388 联调、I2S 音频输出。
  推荐先看页码 `868-901`，重点页 `868 / 870 / 873 / 880 / 882`。

## 本地例程入口

- `display/实验13 TFTLCD（MCU屏）实验`
  优先看 `readme.txt`、`User/main.c`、`User/main1.c`、`User/main2.c`。
  关注点：`MCU屏/FSMC`、LCD 控制线、数据总线、背光与复位脚。
  教材联动建议：先用第五十二章、第五十三章补显示任务，再结合当前屏幕底层驱动。

- `display/实验28 触摸屏实验`
  优先看 `readme.txt`、`User/main.c`、`Drivers/BSP/TOUCH/*`、`Drivers/BSP/LCD/*`。
  关注点：电阻触摸与电容触摸的差异、触摸控制器链路、触摸与显示初始化顺序。
  教材联动建议：主看第四十章；如果界面要显示中文或图片，再串上第五十二、第五十三章。

- `display/实验36 外部SRAM实验`
  优先看 `readme.txt`、`User/main.c`、LCD 相关 BSP 文件。
  关注点：FSMC/FMC、外部 SRAM 容量验证、SRAM 与 LCD 共总线时的资源冲突。
  教材联动建议：它本身更像底层资源例程；如果后续要承载字库、图片或显示缓存，分别串联第五十二章、第五十三章。

- `display/实验40 汉字显示实验`
  优先看 `readme.txt`、`User/main.c`。
  关注点：字库文件、NORFLASH/SD/FATFS、LCD 显示链路。

- `display/实验41 图片显示实验`
  优先看 `readme.txt`、`User/main.c`。
  关注点：图片格式、`PICTURE` 目录、文件系统与 LCD 显示链路。

- `display/实验43 音乐播放器实验`
  优先看 `readme.txt`、`User/main.c`、`User/APP/audioplay.c`、`User/APP/audioplay.h`。
  关注点：WAV 文件约束、`MUSIC` 目录、ES8388、I2S/IIS、DMA、SD/FATFS。

## CubeMX 迁移提醒

- `TFTLCD（MCU屏）`
  先确认是不是 `FSMC/FMC + 并口屏` 方案，而不是普通 SPI 屏。
  检查 `CS / RS(DC) / WR / RD / RST / BL` 和 `D0-D15` 的连接关系。
  不要只移植 LCD 驱动函数，先确认总线和控制脚已经在 `.ioc` 里占好。

- `触摸屏`
  先判断是电阻触摸还是电容触摸。
  电阻触摸更偏 `ADC/GPIO` 复用与校准流程。
  电容触摸更偏 `I2C/SPI + EXTI/RESET` 控制器链路。
  如果触摸和显示共板，初始化顺序和资源复用要一起看。

- `外部 SRAM`
  先确认 `FSMC/FMC` 已启用，再确认 `Bank / 数据宽度 / 时序`。
  如果 SRAM 和 LCD 共用外部总线，优先核对 `NE / NOE / NWE / 地址线 / 数据线` 是否冲突。
  如果 SRAM 用来做字库、图片缓存或显示缓冲区，还要继续检查 `SD/FATFS -> SRAM -> LCD` 数据路径。

## 迁移原则

- 触摸屏和显示屏通常是联动任务，不要把触摸代码和屏幕总线拆开看。
- 汉字和图片显示通常依赖字库、素材和文件系统，不要只迁 LCD 绘图接口。
- 外部 SRAM 更偏底层资源能力，真正价值往往体现在给字体、图片和显示缓冲做承载。
- 音频播放不是单外设移植，至少要同时确认文件源、编解码器、时钟和 DMA。
